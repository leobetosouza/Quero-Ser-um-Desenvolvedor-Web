Quer ser um Desenvolvedor Web?

Você tem que saber o que é web

A Web foi criada por um cara chamado Tim Beners-Lee

Cientista que precisava compartilhar documentos das suas pesquisas e acessar os documentos das pesquisas dos outros cientistas...

A web é uma coleção de documentos linkados

Um documento faz referencia outro(s) documento(s). Esses são os Hiperlinks!

Esses documentos são de Hipertexto

Estruturados através de marcação HTML

Hipertexto + imagens + videos + músicas + planilhas + etc = Hipermídia

Esse documentos ficam armazenados em servidores

Navegadores e servidores se comunicam através do HTTP

HTTP e HTML são as bases da Web

Hypertext Transfer Protocol

Arquitetura Cliente-Servidor

Hey! Servidor, me dá o que tá nessa URL!

http://www.site.com.br/artigos/xxx.html

Ahhh sim! Enjoy! The Internet is for it! :D

Você precisa saber ler os headers HTTP!

Hipertext Markup Language

HTML não é uma línguagem de programação!

Semântica

A ciência do significado!

Fazer um HTML é dizer à máquina o que aquela informação representa!

A Web foi criada pra compartilhar informações!

Ela é o maior sistema de informação que existe!

Com o tempo a informação armazenada ficou grande e complexa

Servidores passaram a ser programáveis...

Quase toda linguagem pode ser usada no server-side

PHP, Java, C#, Python, Perl, Ruby, Scala, etc, etc...

Aprenda pelo menos uma!

Mesmo que você trabalhe com front-end, saber como funciona o back-end te ajuda!

Normalmente a programacão server-side é feita numa arquitetura MVC

  • Model: Camada que entra em contato com os dados, normalmente armazenados num SGBD. Onde fica sua lógica de negócios;
  • View: Camada que mostra como a informação vai ser representada. O front-end;
  • Controller: Groceiramente a ponte entre as outras duas camadas.

Aprenda SQL e pelo menos o básico sobre banco de dados

Tabelas, chaves, SELECT, UPDATE, JOIN, Transactions, etc...

Você não precisa ser um DBA, mas não seja um analfabeto

Quase toda linguagem serve-side tem um bom framework web

Nomes como Rails, Django, Cake, Zend, CodeIgniter, Express, Catalyst, Hibernate são comuns

Mas tente não ser totalmente dependente de um framework

Você sabe aparafusar um parafuso mesmo sem uma chave de fenda elétrica, né?

Aprenda a programar direito!

Lógica, Estruturas de Dados, Design Patterns...

Até outro dia ninguém dava importancia pro cara do HTML

Mas hoje em dia o front-end tá na moda!

Desenvolvimento client-side também tem 3 camadas

  • Estrutura: a marcação HTML;
  • Apresentação: o que diz como o documento é estilizado. Responsabilidade das CSS;
  • Comportamento: como o usuário interage com o documento. O JavaScript.

Estruturar bem seu site é bom pra SEO

Aparecer da maneira certa no Google pode ser a diferença entre sucesso e fracasso do seu site/aplicação

Cascading Style Sheets

Serve pra fazer as coisas ficarem bonitas, ou não.

No principio CSS servia só pra mexer com cores e fonts

Agora ela já invadiu as outras duas camadas

Cuidado! Conteudo criado via CSS não é visto por search engines

Ou seja: não conta pra SEO

CSS3 já permite controlar animações e transições

Essa apresentação usa isso!

E incrivelmente CSS3 é turing complete...

Tenha medo

JavaScript era usado só pra validar formularios e fazer coisas irritantes

Alguns anos atrás tudo mudou

Com PCs e browsers melhores, passou ser possivel fazer grandes processamentos no client-side

Hoje em dia JS é a principal linguagem de programação da web! Você não é um webdev completo se não souber JavaScript direito.

E ele ainda roda no servidor via NodeJS

Inventaram um negócio chamado AJAX

Asynchronous JavaScript and XML

AJAX mudou a web!

XMLHttpRequest (ou xhr para os intímos)

Foi criado pela Microsoft no IE5...

Permite fazer requests HTTP asíncronos

A página não precisa ser recarregada pra você ter conteúdo novo

GMail, Twitter, Facebook, Google Maps e cia

Usam e abusam disso!

Hoje em dia é possível inclusive o servidor enviar mensagens para o front-end

Isso tudo permite que tenhamos Jogos Complexos e grandes apps rodando no browser.

Existem frameworks MVC-like para front-end escritos em JS (como MVVM, MVP e MVW)

Backbone, Angular.js, Knockout.js, Ember

O Model interage com o servidor atravez de APIs

Uma Application Programming Interface especifica como o softwares (ou suas partes) interagem

Em algumas webapps atuais a view se limita a uma API REST

JSON e XML são comumente usados para marcar WEB APIs

Uma vantagem de ter uma API para sua aplicação é que seus dados podem ser usados por sua webapp, aplicativos mobile e até aplicativos desktop

Você basicamente tem um Web Service pra fazer o que quiser sempre.

Então temos o conceito de Web as Platform!

Na Web Platform as aplicações devem estar disponiveis a qualquer momento.

E na Open Web em qualquer lugar!

Mais algumas coisas que você deve saber

Performance Matters!

Seu servidor e seu banco devem responder rápido, mas...

Front-end é ~80% da performance da sua aplicação.

Estude WPO!

Bancos Relacionais são legais...

Mas NoSQL tá na moda! E uma coisa não elimina a outra.

Web Semantics é uma realidade!

RDF, RDFa, SPARQL, Microdata, Turtle, Linked Data

Acessibilidade (e Usabilidade) não é caridade.

Aprenda inglês.

Frequente eventos da área e crie um bom networking

Aprenda um pouco de design.

Não tenha medo do terminal.

Não seja dependente da sua IDE ou editor favorito.

Saiba usar um sistema de controle de versão.

Git e SVN são os mais usados.

Mas aprenda Git e tenha uma conta no GitHub!
